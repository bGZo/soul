  - Vendors and platforms
    Google (Android and Chrome OS)
    Google's Noto fonts project includes the Noto Color Emoji font, which supplies colour glyphs for emoji characters.[132] Chrome OS, through its inclusion of the Noto fonts, supports the emoji set introduced through Unicode 6.2. As of Chrome OS 41, Noto Color Emoji is the default font for most emoji.
  - Android devices support emoji differently depending on the operating system version. Google added native emoji support to Android in July 2013 with Android 4.3,[133] and to the Google Keyboard in November 2013 for devices running Android 4.4 and later.[134] Android 7.0 Nougat added Unicode 9 emoji, skin tone modifiers, and a redesign of many existing emoji.[135]
  - Emoji are also supported by the Google Hangouts application (independent of the keyboard in use), in both Hangouts and SMS modes.[136] Several third-party messaging and keyboard applications (such as IQQI Keyboard) for Android devices[137] provide plugins that allow the use of emoji. Some apps, e.g. WhatsApp, come with Apple emoji for internal use.[clarification needed] With Android 8 (Oreo), Google added a compatibility library that, if included by app developers, makes the latest Noto emoji available on any platform since Android 4.3.[138]
  - Stock Android systems include the Noto glyphs for emoji characters, although individual social media apps may use their own glyphs instead.[139] However, mobile phone vendors HTC and LG deployed variants of NotoColorEmoji.ttf with custom glyphs prior to 2017,[140] and Samsung still does.[141] Some Japanese mobile carriers used to equip branded Android devices with emoji glyphs that were closer to the original ones, but apparently have stopped updating these circa 2015.[clarification needed]
  - Apple 😃
  - Apple first introduced emoji to their desktop operating system with the release of OS X 10.7 Lion, in 2011. Users can view emoji characters sent through email and messaging applications, which are commonly shared by mobile users, as well as any other application. Users can create emoji symbols using the "Characters" special input panel from almost any native application by selecting the "Edit" menu and pulling down to "Special Characters", or by the key combination ⌘ Command+⌥ Option+T. The desktop OS uses the Apple Color Emoji font that was introduced earlier in iOS. This provides users with full color pictographs.[142]
  - The emoji keyboard was first available in Japan with the release of iPhone OS version 2.2 in 2008.[143] The emoji keyboard was not officially made available outside of Japan until iOS version 5.0.[144] From iPhone OS 2.2 through to iOS 4.3.5 (2011), those outside Japan could access the keyboard but had to use a third party app to enable it. The first of such apps was developed by Josh Gare; emoji beginning to be embraced by popular culture outside Japan has been attributed to these apps.[145][146] iOS was updated to support Fitzpatrick skin-tone modifiers with version 8.3.[147]
  - OS X 10.9 Mavericks introduced a dedicated emoji input palette in most text input boxes using the key combination ⌘ Command+Ctrl+Space.[148]
  - Apple has revealed that the "face with tears of joy" is the most popular emoji among English speaking Americans. On second place is the "heart" emoji followed by the "Loudly Crying Face".[149][150]
  - On July 17, 2018, for the World Emoji Day, Apple announced that it will be adding 70 more emoji in its 2018 iOS update, including the long-awaited, red hair, white hair, curly hair and bald emoji.[151][152]
  - On September 12, 2017, Apple announced that the Messages app on the iPhones with Face ID would get "Animoji", which are versions of standard emoji that are custom-animated with the use of facial motion capture to reflect the sender's expressions. These Animoji can also utilize lip sync to appear to speak audio messages recorded by the sender. Apple had created 3D models of all standard emoji prior to its late-2016 OS updates from which the static default 2D graphics had been rendered. A select set of these models are being reused for creating still images and short animations dynamically.
  - With the release of iOS 13, Apple introduced "Memoji" that allows the use of an avatar that a user can use to personalize messages; this feature does not require Face ID.[153]
  - Linux
    Ubuntu 18.04 and Fedora 28 support color emoji by default, using Noto Color Emoji.[154][155] Some Linux distributions require the installation of extra fonts.[156] Color emoji are supported by FreeType and Cairo.[157][158]
  - Microsoft Windows
    An update for Windows 7 and Windows Server 2008 R2 brought a subset of the monochrome Unicode set to those operating systems as part of the Segoe UI Symbol font.[159] As of Windows 8.1 Preview, the Segoe UI Emoji font is included, which supplies full-color pictographs. The plain Segoe UI font lacks emoji characters, whereas Segoe UI Symbol and Segoe UI Emoji include them.
  - Emoji characters are accessed through the onscreen keyboard's "smiley" key.
  - Differently from macOS and iOS, color glyphs are only supplied when the application supports Microsoft's DirectWrite API, and Segoe UI Emoji is explicitly declared, otherwise monochrome glyphs appear.[160] Microsoft's COLR/CPAL format for multi-color fonts such as Segoe UI Emoji is supported by the current versions of several web browsers on Windows (including Firefox, Google Chrome, Internet Explorer and Microsoft Edge), but not by many graphics applications.[125]
  - Windows 10 Anniversary Update added Unicode 9 emoji.[161]
  - Social media platforms
    Facebook and Twitter replace all Unicode emoji used on their websites with their own custom graphics.
  - Prior to October 2017, Facebook had different sets for the main site and for its Messenger service, where only the former provides complete coverage. Messenger now uses Apple emoji on iOS, and the main Facebook set elsewhere.[162] Facebook reactions are only partially compatible with standard emoji.[citation needed]
  - Twitter has released Twemoji, which is their emoji graphics together with a JavaScript library to handle them, under the Creative Commons CC-BY 4.0 license and the MIT open-source license, respectively.[163] Despite this, the Android and iOS Twitter apps use the emoji graphics that are native to the platform they are running on (Apple and Google), instead of the Twemoji graphics.
  - Other emoji font vendors
  - EmojiOne 2.2 logo on the ticket emoji
    EmojiOne version 2.2, an open-source font available under a free content license, supports the full emoji set in color through Unicode Emoji 3.0, i.e. Unicode 9.0. Newer versions of EmojiOne, since renamed JoyPixels,[164] support more recent Unicode Emoji versions, and use a stricter license that disallows the redistribution of vector images, while version 2.x is "no longer supported or distributed".[165] EmojiTwo, an open-source fork of EmojiOne 2.2, aims to add all emoji from 2017 and later.
  - As part of the now-discontinued Firefox OS project, Mozilla developed an emoji font named FxEmojis.[166][167] Mozilla also package a version of Twitter's Twemoji font converted to a COLR/CPAL layered format font, named "Twemoji Mozilla".[168] Older versions of the latter Mozilla project instead packaged the EmojiOne font, as "EmojiOne Mozilla".[169]
  - The font Symbola contains all emoji through version 10.0 as normal monochrome glyphs. Through version 10, Symbola was a public domain font; beginning with version 11 in 2018, Symbola has been copyrighted with a ban on commercial use and derivative works. Other typefaces including a significant number of emoji characters include Noto Emoji, Adobe Source Emoji, and Quivira.
  - Google（Android和Chrome操作系统）
    Google的Noto字体项目包括Noto Color Emoji字体，该字体为Emoji字符提供颜色字形。[132] Chrome操作系统通过包含Noto字体，支持通过Unicode 6.2引入的表情符号集。从Chrome OS 41开始，Noto Color Emoji是大多数表情符号的默认字体。
  - Android设备对表情符号的支持因操作系统版本而异。Google在2013年7月通过Android 4.3 [133]向Android添加了原生表情符号支持，并在2013年11月为运行Android 4.4及更高版本的设备向Google键盘添加了本机表情符号支持。[134] Android 7.0 Nougat添加了Unicode 9表情符号，肤色修饰符，并对许多现有的表情符号进行了重新设计。[135]
  - 在环聊和短信模式下，Google环聊应用程序（与使用的键盘无关）也支持表情符号。[136]若干用于Android设备的第三方消息传递和键盘应用程序（例如IQQI键盘）[137]提供了允许使用表情符号的插件。某些应用程序（例如WhatsApp）附带Apple表情符号供内部使用。[需要澄清]在Android 8（Oreo）上，Google添加了一个兼容性库，如果应用程序开发人员将其包含在内，则可以在Android 4.3以后的任何平台上使用最新的Noto表情符号。[138]
  - 普通的Android系统包括用于表情符号字符的Noto字形，尽管各个社交媒体应用可能会使用自己的字形。[139]但是，手机供应商HTC和LG在2017年前部署了带有自定义字形的NotoColorEmoji.ttf变体，[140]而三星仍然这样做。[141]一些日本移动运营商过去曾为品牌的Android设备配备了与原始表情符号更加接近的表情符号，但显然在2015年左右已停止更新这些表情符号。[需要澄清]
  - 苹果
    苹果在2011年发布了OS X 10.7 Lion，从而将emoji表情引入了其台式机操作系统。用户可以查看通过电子邮件和消息传递应用程序发送的emoji表情字符，移动用户以及任何其他应用程序通常共享它们。用户可以，或者通过组合键创建使用“字符”特殊输入从几乎任何本地应用程序面板中选择“编辑”菜单，下拉到“特殊字符”的表情符号的符号⌘ Command+ ⌥ Option+ T。桌面操作系统使用iOS早期引入的Apple Color Emoji字体。这为用户提供了全彩色的象形文字。[142]
  - Emoji键盘于2008年在iPhone OS 2.2版中在日本首次使用。[143] emoji键盘直到iOS 5.0才在日本以外的地方正式提供。[144]从iPhone OS 2.2到iOS 4.3.5（2011），日本境外的用户可以使用键盘，但必须使用第三方应用程序才能启用键盘。这类应用程序的第一个由Josh Gare开发；表情符号开始被日本以外的流行文化所接受，这归因于这些应用程序。[145] [146] iOS已更新，以支持8.3版的Fitzpatrick肤色修改器。[147]
  - OS X 10.9小牛使用组合键引入在大多数文本输入框的专用表情符号调色板输入⌘ Command+ Ctrl+ Space。[148]
  - 苹果公司透露，“带着喜悦的眼泪的脸”是说英语的美国人中最受欢迎的表情符号。第二位是“心脏”表情符号，其次是“大哭泣的脸”。[149] [150]
  - 在2018年7月17日的世界表情符号日，苹果宣布将在其2018 iOS更新中添加70多个表情符号，包括期待已久的红头发，白头发，卷发和秃头表情符号。[151] [152]
  - 2017年9月12日，Apple宣布具有Face ID的iPhone上的Messages应用程序将获得“ Animoji”，这是标准表情符号的版本，并使用面部动作捕捉来自定义动画，以反映发件人的表情。这些Animoji也可以利用口型同步来朗读发件人记录的音频消息。Apple在2016年后期的OS更新之前已经创建了所有标准表情符号的3D模型，并从中渲染了静态默认2D图形。这些模型中的一组选择被重用于动态创建静态图像和短动画。
  - 随着iOS 13的发布，Apple引入了“ Memoji”，该表情符号允许用户使用化身来个性化消息。此功能不需要人脸ID。[153]

    EmojiOne 2.2版是一种在免费内容许可下提供的开源字体，它支持通过Unicode Emoji 3.0（即Unicode 9.0）以彩色设置的完整表情符号。EmojiOne的较新版本（已重命名为JoyPixels [164]）支持较新的Unicode Emoji版本，并使用更严格的许可，禁止重新分发矢量图像，而2.x版本“不再受支持或分发”。[165] EmojiTwo是EmojiOne 2.2的一个开源分支，旨在添加2017年及以后的所有emoji。
